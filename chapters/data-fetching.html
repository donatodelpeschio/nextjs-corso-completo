Data Fetching Avanzato

Fetch API Estesa

Next.js estende fetch con caching e revalidation automatici.

Caching Automatico

// Default: cached (SSG)
const data = await fetch('https://api.example.com/posts')

// No cache (SSR)
const data = await fetch('https://api.example.com/posts', {
  cache: 'no-store'
})

// Revalidate ogni 60 sec (ISR)
const data = await fetch('https://api.example.com/posts', {
  next: { revalidate: 60 }
})

Tag-based Revalidation

// Tagga la fetch
const posts = await fetch('https://api.example.com/posts', {
  next: { tags: ['posts'] }
})

// Poi invalida per tag
import { revalidateTag } from 'next/cache'
revalidateTag('posts')

Server Actions (React 19+)

// app/actions.js
'use server'

export async function createPost(formData) {
  const title = formData.get('title')
  
  await db.posts.create({
    data: { title }
  })
  
  revalidatePath('/blog')
}

// app/blog/new/page.js
import { createPost } from '../actions'

export default function NewPost() {
  return (
    <form action={createPost}>
      <input name="title" />
      <button type="submit">Crea</button>
    </form>
  )
}
🔥 Server Actions

Nuova feature React 19: chiama funzioni server direttamente da form o eventi, senza API routes!

Parallel Data Fetching

// ❌ Sequential (lento)
const user = await fetch('/api/user')
const posts = await fetch('/api/posts')

// ✅ Parallel (veloce)
const [user, posts] = await Promise.all([
  fetch('/api/user'),
  fetch('/api/posts')
])
