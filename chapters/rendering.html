Strategie di Rendering

I Tre Pilastri del Rendering in Next.js 15+

Next.js ti dà 3 modi fondamentali per renderizzare le pagine con performance e freschezza dei dati ottimali.

1. Static Site Generation (SSG) ⚡

Quando: Build time
Cosa: HTML pre-generato una volta
Ideale per: Contenuti stabili

// app/about/page.js
export default function About() {
  return <h1>Chi Siamo - Pagina Statica</h1>
}
// ✅ Generato al build, veloce come un razzo!

2. Server-Side Rendering (SSR) 🔄

Quando: Ogni richiesta
Cosa: HTML generato on-demand
Ideale per: Dati real-time

async function getData() {
  const res = await fetch('https://api.example.com/data', {
    cache: 'no-store' // SSR!
  })
  return res.json()
}

3. Incremental Static Regeneration (ISR) 🔄⚡

Quando: Build + revalidation periodica
Cosa: Statico che si aggiorna
Ideale per: Meglio di entrambi!

const data = await fetch('https://api.example.com/posts', {
  next: { revalidate: 3600 } // Aggiorna ogni ora
})
🎯 Come Scegliere?
  • Mai cambia → SSG
  • Cambia ogni ora/giorno → ISR
  • Sempre diverso → SSR
