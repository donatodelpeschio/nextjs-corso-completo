Pages Router vs App Router

Due Paradigmi Diversi

Aspetto Pages Router App Router
Introdotto Next.js 1 (2016) Next.js 13 (2022)
Stato (2026) Maintenance mode ✅ Raccomandato
Cartella pages/ app/
Componenti Default Client Server
Data Fetching getServerSideProps async/await
React React 18 React 19+
Params (Next.js 15+) Sync Async (Promise)
💡 Il Futuro è App Router

Nel 2026, Pages Router è in maintenance mode. Tutte le nuove features (Server Components, Streaming, PPR) sono solo in App Router.

Confronto Codice

Metadata (SEO)

// PAGES ROUTER
import Head from 'next/head'

export default function Page() {
  return (
    <>
      <Head>
        <title>Titolo</title>
        <meta name="description" content="..." />
      </Head>
      <div>Contenuto</div>
    </>
  )
}

// APP ROUTER
export const metadata = {
  title: 'Titolo',
  description: '...'
}

export default function Page() {
  return <div>Contenuto</div>
}

Data Fetching

// PAGES ROUTER
export async function getServerSideProps() {
  const data = await fetch('...')
  return { props: { data } }
}

export default function Page({ data }) {
  return <div>{data}</div>
}

// APP ROUTER
async function getData() {
  return await fetch('...')
}

export default async function Page() {
  const data = await getData()
  return <div>{data}</div>
}

Quando Usare Cosa?

✅ Usa App Router se:
  • Nuovo progetto (2026)
  • Vuoi React 19 e Server Components
  • Vuoi performance massime
  • Vuoi le ultime features
⚠️ Usa Pages Router solo se:
  • Progetto esistente legacy
  • Dipendenze incompatibili con Server Components
  • Team non pronto al cambiamento

Migrazione

Puoi usare entrambi insieme temporaneamente:

my-app/
├── pages/              ← Route legacy
│   └── old-page.js
└── app/                ← Nuove route
    └── new-page/
        └── page.js
