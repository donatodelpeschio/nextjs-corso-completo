Server Components

La Rivoluzione di React 19

I React Server Components sono la feature più rivoluzionaria dell'App Router.

🎯 Concetto Chiave

Nell'App Router, ogni componente è un Server Component per default. Eseguono sul server, non inviano JavaScript al browser.

Server Component = Default

// app/posts/page.js - SERVER COMPONENT
async function getPosts() {
  const res = await fetch('https://api.example.com/posts')
  return res.json()
}

export default async function Posts() {
  const posts = await getPosts()
  
  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>{post.title}</article>
      ))}
    </div>
  )
}

// ✅ Esegue sul server
// ✅ Può essere async
// ✅ Accesso diretto al database
// ❌ NO useState, useEffect, onClick

Vantaggi

  • Zero JavaScript al client - bundle più piccolo
  • Accesso diretto ai dati - DB, filesystem, API private
  • Sicurezza - segreti restano sul server
  • Performance - elaborazione pesante sul server

Cosa PUOI fare

// ✅ Fetch dati
const data = await fetch('...')

// ✅ Leggere database
import { db } from '@/lib/db'
const users = await db.users.findMany()

// ✅ Leggere file
import fs from 'fs/promises'
const content = await fs.readFile('data.json')

// ✅ Usare variabili d'ambiente
const key = process.env.SECRET_KEY

Cosa NON puoi fare

// ❌ Hooks
const [count, setCount] = useState(0)  // Errore!

// ❌ Event handlers
<button onClick={() => {}}  // Errore!

// ❌ Browser APIs
window.localStorage  // Errore!

Usa un Client Component (prossimo capitolo)

Accesso Diretto al Database

// app/users/page.js
import { db } from '@/lib/db'

export default async function Users() {
  // Fetch DIRETTO dal database!
  const users = await db.user.findMany({
    orderBy: { createdAt: 'desc' }
  })
  
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}
